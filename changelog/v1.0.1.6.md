## [**1.0.1.6**] - 2025-12-29

Major Spec Kit Memory system upgrade with 7 new tools, checkpoint support, and 30+ metadata fields for advanced filtering.

---

### New

1. **7 New MCP Tools**:
   - `memory_health` — Database statistics and performance metrics
   - `memory_delete` — Batch delete by spec folder or title pattern
   - `memory_update` — Modify existing memories
   - `memory_validate` — Validate spec folders against templates
   - `memory_index_scan` — Force reindex spec folders
   - `checkpoint_create` / `checkpoint_list` / `checkpoint_delete` — Database state snapshots
2. **30+ Metadata Fields** — `specFolder`, `level`, `status`, `tags[]`, `priority`, `git_branch`, `author`, `related_specs[]`, `dependencies[]`, `blockers[]`
3. **Smart Ranking** — Time-decay scoring with configurable weights
4. **Auto-Anchors** — Automatic extraction of heading-based anchors for targeted queries
5. **Embedding Providers** — Support for Voyage AI (1024d) and OpenAI (1536d)
6. **Multi-Concept Search** — Search by multiple concepts (OR logic)
7. **Advanced Filters** — By spec folder, level, status, tags, priority, date ranges
8. **Trigger Phrase Limits** — Configurable max per memory (prevents bloat)

---

### Changed

1. Database schema updated with 30+ new columns
2. Memory index now includes heading-based anchors
3. Constitutional memories separated from specs
4. Smart scoring replaces pure semantic matching
5. `generate-context.js` outputs all 30+ metadata fields

---

### Fixed

1. Migration scripts for schema changes (v15 → v16)
2. Backward compatibility for existing memories
3. Error handling for missing metadata fields
