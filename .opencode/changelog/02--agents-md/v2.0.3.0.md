## [**2.0.3.0**] - 2026-02-20

**Fix invalid tool names in Gemini CLI agents** -- Five agent files failed to load due to two incorrect tool name strings (`edit_file`, `web_search`) in YAML frontmatter `tools:` lists. Replaced with valid Gemini CLI registry names (`replace`, `google_web_search`).

---

## Highlights

### Category: Bugfix

- **`edit_file` → `replace`**: Corrected in 5 agents (debug, handover, research, speckit, write) — `edit_file` does not exist in the Gemini CLI tool registry; the correct constant is `EDIT_TOOL_NAME = "replace"`
- **`web_search` → `google_web_search`**: Corrected in 2 agents (research, write) — `web_search` does not exist; the correct constant is `WEB_SEARCH_TOOL_NAME = "google_web_search"`

### Category: Verification

- **Working agents unmodified**: `context.md`, `orchestrate.md`, and `review.md` only use valid tool names and required no changes
- **Zero invalid names remaining**: Post-edit grep confirms no `edit_file` or `web_search` entries across any agent file
