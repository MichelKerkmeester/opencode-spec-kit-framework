## Highlights

### Category: Features

- **8 Gemini-native agents**: orchestrate, context, debug, research, review, speckit, write, handover -- all with Gemini CLI YAML frontmatter (`kind`, `tools` array, `max_turns`, `timeout_mins`)
- **Role-tuned max_turns**: orchestrate=25, debug/research/speckit=20, review/write=15, context/handover=10 (calibrated for 400K effective token window)
- **Role-tuned timeout_mins**: orchestrate=15, debug/research=15, speckit/write=10, context/review/handover=5
- **Routing-optimized descriptions**: Each agent description rewritten to help Gemini's routing logic decide when to invoke -- includes purpose, trigger conditions, output type, and boundary constraints

### Category: Architecture

- **Token budget analysis**: All agents validated against 400K effective context limit (gemini-3.1-pro has 1M window, degrades beyond 400K). Per-agent budgets calculated: system prompt (3-13K) + overhead (15K) + per-turn cost (5-15K) = safe margins for all configured max_turns
- **Tool scoping**: Tools matched to role -- context/review are read-only (`read_file`, `grep_search`, `list_directory`); research/write include `web_search`; handover excludes `run_shell_command`; orchestrate has `read_file` only

### Category: Documentation

- **Path convention aligned**: All agent bodies reference `.gemini/agents/*.md` as canonical path
- **Template compliance**: All 8 agents validated against workflows-documentation agent template -- required sections (CORE WORKFLOW, CAPABILITY SCAN, OUTPUT VERIFICATION, ANTI-PATTERNS, RELATED RESOURCES, SUMMARY) confirmed present
