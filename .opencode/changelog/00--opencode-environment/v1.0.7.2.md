## [**1.0.7.2**] - 2026-01-23

**Consolidated Setup Prompts** — Reduces user interactions from 3-5 round-trips to just 1 by consolidating all setup questions into a single prompt across 7 SpecKit commands. Also removes outdated model version references like GPT-4/o1 and Pro/Ultra.

> Spec folder: `system-spec-kit/consolidated-prompts`

---

## Highlights

### Single Consolidated Prompt
- **All 7 commands** now ask all questions in ONE prompt instead of multiple phases: research, plan, implement, complete, debug, resume, handover
- **Before**: 3-5 separate interactions (spec folder → execution mode → dispatch mode → memory context)
- **After**: Single consolidated prompt with Q0-Q5, reply with "A, B, A" format

### User-Facing Model Selection
- **Q2 AI Model options** in debug.md simplified: `B) Gemini - Google models (Pro/Ultra)` → `B) Gemini - Google`, `C) Codex - OpenAI models (GPT-4/o1)` → `C) Codex - OpenAI`

### Dispatch Mode Descriptions
- **Mode descriptions** made model-agnostic across all YAML configs: `1 Opus orchestrator + 2 Sonnet parallel workers` → `1 orchestrator + 2 parallel workers`, `1 Opus orchestrator + 3 Sonnet parallel workers` → `1 orchestrator + 3 parallel workers`

## Files Changed
- 7 command files (research.md, plan.md, implement.md, complete.md, debug.md, resume.md, handover.md)
- 10 YAML config files (spec_kit_debug_auto.yaml, spec_kit_debug_confirm.yaml, spec_kit_complete_auto.yaml, spec_kit_complete_confirm.yaml, spec_kit_plan_auto.yaml, spec_kit_plan_confirm.yaml, spec_kit_implement_auto.yaml, spec_kit_implement_confirm.yaml, spec_kit_research_auto.yaml, spec_kit_research_confirm.yaml)

## Upgrade
No action required. Pull latest to get consolidated prompts and model-agnostic naming.
