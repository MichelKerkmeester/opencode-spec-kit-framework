## [**1.0.7.1**] - 2026-01-23

**Multi-Agent Dispatch for SpecKit Commands** — Adds user-selectable A/B/C dispatch modes to 5 spec_kit work-execution commands, enabling 1 Opus orchestrator + 2-3 Sonnet parallel workers with Coordinator/Worker mode instructions in agent files and multi_agent_config sections in 12 YAML configs.

> Spec folder: `system-spec-kit/multi-agent-dispatch`

---

## Highlights

### Multi-Agent Dispatch (A/B/C Selection)
- **Dispatch Mode Phase**: New mandatory phase added to 5 commands (`/spec_kit:complete`, `/spec_kit:plan`, `/spec_kit:implement`, `/spec_kit:research`, `/spec_kit:debug`)
- **Option A**: Single Agent — Execute with one Opus agent (default)
- **Option B**: Multi-Agent (1+2) — 1 Opus orchestrator + 2 Sonnet workers
- **Option C**: Multi-Agent (1+3) — 1 Opus orchestrator + 3 Sonnet workers
- **Coordinator Mode**: Added to `research.md` (Section 9) and `debug.md` (Section 4) with Worker Output Validation checklist and Contradiction Resolution Protocol
- **Worker Mode**: Added to `research.md` (Section 10) and `debug.md` (Section 5) with focused domain execution, structured JSON output, and worker roles table

### Command Phase Structure
- **complete.md**: Phase 2.5 → Phase 3 (Research), new Phase 4 (Dispatch), Phase 3 → Phase 5 (Memory)
- **plan.md**: New Phase 3 (Dispatch), Phase 3 → Phase 4 (Memory)
- **implement.md**: New Phase 3 (Dispatch), Phase 3 → Phase 4 (Memory)
- **research.md**: New Phase 3 (Dispatch), Phase 3 → Phase 4 (Prior Work), Phase 4 → Phase 5 (Memory)

### YAML Configuration
- **12 YAML configs** updated with `multi_agent_config` section: `spec_kit_research_auto.yaml`, `spec_kit_research_confirm.yaml`, `spec_kit_debug_auto.yaml`, `spec_kit_debug_confirm.yaml`, `spec_kit_complete_auto.yaml`, `spec_kit_complete_confirm.yaml`, `spec_kit_plan_auto.yaml`, `spec_kit_plan_confirm.yaml`, `spec_kit_implement_auto.yaml`, `spec_kit_implement_confirm.yaml`
- **Worker Definitions**: Research (codebase_explorer, external_researcher, technical_analyzer), Debug (call_path_tracer, pattern_searcher, edge_case_hunter), Complete/Plan/Implement (architecture_explorer, feature_explorer, dependency_explorer)

### Fixed
- **Phase Numbering**: All commands now use sequential integers (no 2.5 or 3.5)
- **YAML Phase References**: `phase_2_5_research` → `phase_3_research` in complete YAMLs

## Files Changed
- 5 command files (complete.md, plan.md, implement.md, research.md, debug.md)
- 12 YAML configuration files

## Upgrade
No action required. Pull latest to get multi-agent dispatch. Commands will now prompt for dispatch mode (A/B/C) before execution.
