## Highlights

### Critical Import Naming Fixes

- **getDatabasePath mismatch** — `getDatabasePath` → `get_database_path` in `mcp_server/context-server.js:1454` (health check response) and `mcp_server/lib/vector-index.js:100` (database path resolution)
- **validateFilePath mismatch** — `validateFilePath` → `validate_file_path` in `mcp_server/context-server.js:55` and `mcp_server/lib/vector-index.js:13`
- **Error handler mismatches** — `isTransientError` / `userFriendlyError` → `is_transient_error` / `user_friendly_error` in `mcp_server/context-server.js:214`
- **escapeRegex mismatch** — `escapeRegex` → `escape_regex` in `mcp_server/lib/trigger-matcher.js:7` and `mcp_server/lib/memory-parser.js:9`

**Root Cause**: During snake_case refactoring, exports in source modules (`shared/utils.js`, `lib/errors.js`) were renamed but imports in consuming files retained camelCase names. Fixed using import aliasing: `const { snake_case: camelCase } = require(...)`
