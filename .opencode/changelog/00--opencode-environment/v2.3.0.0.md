## Highlights

### Category: Features

- **`.gemini/` directory structure**: Complete Gemini CLI runtime adapter with `agents/`, `commands/`, `skills/`, `specs` symlink, and `settings.json`
- **8 Gemini-native agents**: orchestrate, context, debug, research, review, speckit, write, handover -- each with Gemini YAML frontmatter (`kind: local`, `tools:` array, `max_turns`, `timeout_mins`), routing-optimized descriptions, and 1:1 OpenCode body content
- **19 TOML command wrappers**: All OpenCode commands wrapped in Gemini's `.toml` format using `@{file}` injection to preserve single source of truth -- `spec_kit/` (7), `memory/` (5), `create/` (6), root (1)
- **10 skill symlinks**: All OpenCode skills symlinked into `.gemini/skills/` for direct Gemini CLI access
- **`GEMINI.md`**: Symlink to `AGENTS.md` providing project-level behavioral instructions to Gemini CLI
- **3 MCP servers configured**: `sequential_thinking`, `spec_kit_memory`, `code_mode` -- all with `trust: true` for confirmation-free tool execution

### Category: Architecture

- **4-provider runtime matrix**: Framework now supports Copilot, ChatGPT, Claude, and Gemini CLI as first-class runtimes with consistent agent/command/skill access
- **Token budget optimization**: All agents calibrated for gemini-3.1-pro 400K effective window -- role-tuned `max_turns` (10-25) and `timeout_mins` (5-15) based on per-agent token budget analysis
- **Tool scoping per role**: Context/review read-only (`read_file`, `grep_search`, `list_directory`); research/write include `web_search`; handover excludes `run_shell_command`; orchestrate minimal (`read_file` only)
- **TOML `@{file}` injection pattern**: Commands reference OpenCode `.md` source via relative paths, eliminating content duplication while adapting to Gemini's TOML format requirement

### Category: Documentation

- **AGENTS.md ยง7 extended**: Runtime resolution table now includes Gemini CLI row
- **Path conventions aligned**: All Gemini agent bodies reference `.gemini/agents/*.md`
- **Template compliance validated**: All 8 agents pass workflows-documentation agent template checklist (CORE WORKFLOW, CAPABILITY SCAN, OUTPUT VERIFICATION, ANTI-PATTERNS, RELATED RESOURCES, SUMMARY)
- **Spec folder 023**: Full Level 2 spec folder documenting planning, implementation, and verification

1. **Pull latest** to get the new `.gemini/` directory
2. **Install Gemini CLI** if not already installed
3. **Verify** with `gemini agents list` and `/mcp` inside a Gemini CLI session
4. No migration required for existing runtimes -- all changes are additive
