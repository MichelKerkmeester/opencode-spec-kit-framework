## [**1.0.7.0**] - 2026-02-16

**Remove emoji enforcement from documentation** — Removed H2 emoji requirements from all document types. Emojis are no longer enforced in H2 headings, TOC entries or validation checks. Stripped emojis from all existing H2 headings across templates, references, core docs and test fixtures.

> Spec folder: `specs/002-commands-and-skills/003-workflows-documentation/005-remove-emojis-from-docs/`

## Highlights

### Changed

- **`assets/template_rules.json`** — Set `h2_emoji_required: false` for all 7 document types (readme, skill, command, install_guide, reference, asset, agent). Removed `missing_h2_emoji` and `toc_missing_emoji` from blocking errors. Removed `add_h2_emoji` and `fix_toc_emoji` auto-fix rules
- **`scripts/validate_document.py`** — Removed TOC emoji presence check from `validate_toc()`. H2 emoji validation now skipped when `h2_emoji_required` is false
- **`scripts/extract_structure.py`** — Cleared `EMOJI_REQUIRED_TYPES` set (no document types require emojis). Removed `h2_emoji` checklist items from SKILL, README, asset and reference checklists. Updated DQI style scoring to evaluate number + ALL CAPS only (removed emoji factor)
- **`SKILL.md`** — Stripped emojis from 8 H2 headings. Removed "Emoji Usage Rules" and "H2 Emoji Enforcement by Document Type" subsections from Section 4 RULES
- **`README.md`** — Stripped emojis from 8 TOC entries and 8 H2 headings. Removed emoji error row from troubleshooting table
- **`references/core_standards.md`** — Stripped emojis from all H2 headings. Removed entire Section 8 "EMOJI USAGE RULES" (renumbered Section 9 to 8). Removed "Emojis Allowed" column from enforcement table. Removed "H2 Emoji Violations (BLOCKING)" subsection. Updated code examples throughout
- **`references/validation.md`** — Stripped emojis from 10 H2 headings. Removed emoji check rows from Stage 1 table. Updated DQI style score references from "number+emoji+CAPS" to "number+CAPS"
