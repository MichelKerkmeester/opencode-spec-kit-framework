#### Implementation-Summary Template Improvements

- **Opening hook guidance** added to all levels. Templates now guide writers to lead with impact in 2-4 sentences.
- **Feature subsection pattern** introduced. Level 3/3+ templates use `### [Feature Name]` subsections per feature instead of file tables.
- **Files Changed table removed for Level 3/3+** - Narrative carries the summary at higher documentation levels. Level 1-2 retains the table.
- **Honest verification guidance** added. Voice guide now says: "Be honest. Show failures alongside passes."
- **Numbered limitations format** standardized. All levels use `1. **[Limitation]** [Specific detail with workaround.]`
- **Delivery stages guidance** added for Level 3+ (testing, rollout, verification).
- **Spec 136 added as reference** in voice guide comments.

### Fixed

- **Spec 137 implementation-summary.md** - Updated stale HVR_REFERENCE path (lines 5, 144) to point to new `references/` location.
