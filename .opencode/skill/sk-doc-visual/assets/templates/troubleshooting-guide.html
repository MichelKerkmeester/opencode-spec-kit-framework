02 // P0 Critical Fixes

P0 CRITICAL FIX-001

Form submit handler silently fails on validation error

~45 min

Impact: Users click submit, nothing happens. No error shown, no feedback. Affects 100% of submissions with invalid fields.

LOCATION: src/components/contact-form.js lines 138-155

FLOW DIAGRAM

 User clicks Submit | preventDefault() | validateForm() ──── throws ValidationError | | (success) catch(e) { } <-- EMPTY: no UI update | | sendFormData() *** SILENT FAIL *** | showSuccess()

FIX STEPS

1

Open contact-form.js, locate handleSubmit at line 138.

2

Replace empty catch block with error state propagation and user notification.

3

Wire the aria-live="assertive" region to receive the error message (see FIX-002).

BEFORE

async handleSubmit(e) { e.preventDefault(); try { const data = this.validateForm(); await this.sendFormData(data); this.showSuccess(); } catch (err) {
 // TODO: handle errors }
}

AFTER

async handleSubmit(e) { e.preventDefault(); try { const data = this.validateForm(); await this.sendFormData(data); this.showSuccess(); } catch (err) { this.setErrorState(err.fields); this.announceError(err.message); this.focusFirstInvalid(); }
}

VERIFY

Submit with empty required field. Confirm error appears, invalid field focused, screen readers announce via live region.

03 // P1 High Priority Fixes

P1 HIGH FIX-003

Tab order skips email field

Email input has tabindex="-1", removing it from tab sequence. Keyboard users cannot reach the field. WCAG 2.1 SC 2.1.1 violation.

DECISION GUIDE

  • Option A: Remove tabindex entirely

    Recommended. Natural DOM order handles tab sequence.

    • If field order matches visual order → use this
    • If no JS depends on tabindex → use this
  • Option B: Set tabindex="0"

    Only if custom focus management requires explicit values.

    • If dynamic field ordering is used → consider this
P1 HIGH FIX-005

Error text color fails WCAG AA contrast

Error messages use #ff6b6b on white (3.2:1). WCAG AA requires 4.5:1 for normal text.

BEFORE (3.2:1)

.field-error { color: #ff6b6b; }

AFTER (7.1:1)

.field-error { color: #b91c1c; }
