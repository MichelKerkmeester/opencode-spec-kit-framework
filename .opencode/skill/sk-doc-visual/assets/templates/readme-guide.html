01 // Overview

Eliminating AI amnesia via continuous searchable history.

What is This?

AI coding assistants are powerful, but they have amnesia. Every session starts from zero. You explain your auth system Monday. By Wednesday, it's a blank slate. Architectural decisions? Lost in chat history. Documentation? "I'll do it later" (you won't).

Two custom-built systems fix this: a spec-kit documentation framework and a cognitive memory MCP server that turn stateless AI sessions into a continuous, searchable development history. Purpose-built for AI-assisted development, not a wrapper around existing tools. Developers using AI assistants who are tired of re-explaining context every session and losing decisions to chat history.

Recent Platform Highlights

Hybrid RAG Fusion (Spec 138)

The memory engine activates three retrieval channels simultaneously (Vector, BM25, FTS5) and fuses results via Reciprocal Rank Fusion.

Frontmatter Normalization

Managed metadata standardized across 81 templates so dashboard titles are unique, parser compatibility accepts contextType, and reindex scripts keep index consistent.

Gemini CLI & Docs Search

Gemini CLI is the 4th runtime. Spec documents are indexed and searchable. 473 anchor tags across 74 READMEs enable section-level retrieval with ~93% token savings.

05 // Memory Engine

Remember everything. Surface what matters. Keep it private.

25 MCP tools across 7 architectural layers: 3-source indexing, 7-intent retrieval routing, schema v15 metadata and document-type scoring. The Unified Context Engine adds a 3-channel hybrid retrieval pipeline with RRF fusion, MMR diversity pruning, confidence gating and causal graph enrichment.

3-Source Discovery Pipeline

Source Weight Path

Document-Type Scoring

Cognitive Features

Attention-Based Decay

Memory relevance decays using recency × frequency × importance. Constitutional tier (1.0) never decays, Normal (0.8) fades gradually, Temporary (0.6) fades fast.

Spreading Activation

When a primary memory is activated, semantically related memories receive a 0.35 score boost (e.g., auth triggers JWT and OAuth setup).

Hybrid Search Architecture

Three channels fuse via RRF: Vector (semantic similarity), BM25 (technical term frequency), and FTS5 (exact substring matching).

Causal Memory Graph

Every decision has a lineage. The causal graph tracks 6 relationship types.

causedJWT decision → token refresh
derived_fromrate limiter ← load test results
supportsbenchmarks → caching decision
supersedesv2 auth flow → v1 auth flow
enabledOAuth2 setup → social login
contradictsstateless ↔ session storage

09 // Gate System

Every request passes through 3 mandatory gates before the AI touches a single file. Enforced, not suggested. No exceptions.

Request ──► Gate 1 (SOFT) ──► Gate 2 (REQUIRED) ──► Gate 3 (HARD) ──► Execute

Gate 1: Understanding

Soft Block

Dual-threshold validation: READINESS = (confidence >= 0.70) AND (uncertainty <= 0.35). Surfaces relevant memories via trigger matching before you ask.

Gate 2: Skill Routing

Required

Runs skill_advisor.py against keywords. Confidence >= 0.8 mandates loading to ensure domain expertise is present.

Gate 3: Spec Folder

Hard Block

No file changes without answering: Use existing? Create new? Update related? Skip? Phase folder?

10 // Code Mode MCP

98.7% context token savings.

External tool integration via TypeScript execution. Instead of loading all tool definitions into context upfront, Code Mode uses progressive disclosure.

Progressive Tool Disclosure

typescript

search_tools({ task_description: "webflow site management" })

// 2. Get parameter details

// 3. Execute with TypeScript
call_tool_chain({ code: "await webflow.webflow_sites_list({})" })

14 // FAQ

Q: Do I need API keys to use the memory system?

A: No. HuggingFace Local runs on your machine with no API keys needed. It's the default fallback. Voyage AI and OpenAI are optional upgrades for better retrieval quality.

Q: How is this different from Cursor's memory or Copilot's context?

A: Those are session-scoped. This system persists across sessions, models and even projects. It uses causal graphs to trace decision lineage and cognitive tiers to prioritize relevance. It uses ANCHOR format for 93% token savings. It's also local-first. Your code stays on your machine.

Q: Is this overkill for solo developers?

A: You might think so, until you lose 3 hours re-debugging an issue you already solved last month. Solo developers benefit more because there's no team to ask "hey, why did we do it this way?" The memory system is your institutional knowledge.

Q: Can I switch embedding providers without losing data?

A: Each provider uses a separate SQLite database file, so switching is safe. Your old embeddings remain intact. Re-index with memory_index_scan if you want to regenerate embeddings with a new provider.
