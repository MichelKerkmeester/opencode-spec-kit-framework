What Was Built

The memory engine moves from opaque retrieval to traceable, intent-aware retrieval. Results carry a typed trace, routing adapts by artifact type, and fusion weights adjust by task intent. The index now records append-only mutation history and captures richer operational telemetry.

Typed Retrieval Trace

Every hit can expose stage-level decisions (candidate selection, filter, fusion, rerank, fallback) with score and timing context.

Artifact-Aware Routing

Specs, plans, decisions, code, config, README, memory, scratch and research can each route to deterministic weighting strategies.

Adaptive Hybrid Fusion

Seven intent profiles tune retrieval emphasis with an explicit opt-out flag path for deterministic fallback.

Ledger + Telemetry

Mutation events become tamper-evident and telemetry captures latency, mode selection, fallback signals and quality proxies.

Key Decisions

Decision Why
Adaptive fusion uses opt-out semantics Default behavior stays adaptive unless explicitly disabled for deterministic comparison.
Append-only ledger enforced at DB layer Trigger-level guarantees are stronger than application-only mutation controls.
Deterministic artifact-routing weights Transparent and auditable retrieval behavior beats opaque adaptive routing for governance.
Telemetry isolated to evaluation lane Prevents synthetic closure metrics from being represented as live production telemetry.
