# CSS Quick Reference

One-page cheat sheet for CSS patterns including Webflow tokens, state machines, form validation, and accessibility.

---

## 1. ðŸŽ¨ WEBFLOW TOKEN NAMING

> **IMPORTANT:** This naming convention is Webflow Designer-specific. These tokens are auto-generated by Webflow's design system and exist in the published CSS. This section documents the pattern for understanding existing code - do NOT create new custom properties using this convention.

### Token Pattern

```
--_category---subcategory--variant
```

| Part          | Delimiter | Example                   |
| ------------- | --------- | ------------------------- |
| Category      | `--_`     | `--_color-tokens`         |
| Subcategory   | `---`     | `---input-border`         |
| Variant       | `--`      | `--negative`              |

### Common Tokens (Read-Only Reference)

```css
/* Input backgrounds */
var(--_color-tokens---input-bg--enabled)
var(--_color-tokens---input-bg--negative)
var(--_color-tokens---input-bg--positive)

/* Input borders */
var(--_color-tokens---input-border--enabled)
var(--_color-tokens---input-border--negative)

/* Content colors */
var(--_color-tokens---input-content--negative)
var(--_color-tokens---content-neutral--dark)

/* State colors */
var(--_color-tokens---state--focused)
var(--_color-tokens---state--warning)
var(--_color-tokens---state--success)

/* Border colors */
var(--_color-tokens---border-neutral--dark)
```

## 2. ðŸ”„ STATE MACHINE USAGE

### Pattern

Webflow's state tokens use `--_state---` prefix with `on`/`off` suffixes:

```css
var(--_state---component--on)
var(--_state---component--off)
```

### Usage in CSS

```css
/* Toggle visibility based on Webflow interaction state */
.element {
  display: var(--_state---menu--on, none);
}

.element-alt {
  display: var(--_state---menu--off, block);
}
```

## 3. âœ… FORM VALIDATION CLASSES

### Classes

| Class                 | Purpose                  | Applied To           |
| --------------------- | ------------------------ | -------------------- |
| `.validation-invalid` | Invalid/error state      | `[data-form-field]`  |
| `.validation-valid`   | Valid/success state      | `[data-form-field]`  |

### CSS Selectors

```css
/* Invalid state - text input */
[data-form-field].validation-invalid input.input[data-form-input] {
  border-color: var(--_color-tokens---input-border--negative);
  background-color: var(--_color-tokens---input-bg--negative);
}

/* Invalid state - custom select */
[data-form-field].validation-invalid [data-select="input"] {
  border-color: var(--_color-tokens---input-border--negative);
  background-color: var(--_color-tokens---input-bg--negative);
}

/* Valid state - show success icon */
[data-form-field].validation-valid #input-success {
  display: flex;
  opacity: 1;
}

/* Helper text colors */
[data-form-field].validation-invalid .input--helper-w {
  color: var(--_color-tokens---state--warning);
}

[data-form-field].validation-valid .input--helper-w {
  color: var(--_color-tokens---state--success);
}
```

### JavaScript Usage

```javascript
// Add validation state
field.classList.add('validation-invalid');

// Remove validation state
field.classList.remove('validation-invalid');

// Toggle states
field.classList.remove('validation-invalid');
field.classList.add('validation-valid');
```

## 4. â™¿ REDUCED MOTION (MANDATORY)

### Requirement

All animations MUST respect `prefers-reduced-motion`. This is a hard requirement for accessibility.

### CSS Pattern

```css
/* Default animation */
.element {
  transition: transform 0.3s ease-out;
}

/* Disable for reduced motion preference */
@media (prefers-reduced-motion: reduce) {
  .element {
    transition: none;
  }
}
```

### JavaScript Pattern

```javascript
// Check preference
const PREFERS_REDUCED_MOTION = window.matchMedia(
  '(prefers-reduced-motion: reduce)'
).matches;

// Use in animation logic
if (!PREFERS_REDUCED_MOTION) {
  animate(element, { opacity: [0, 1], y: [20, 0] });
}

// Alternative: instant transition
if (PREFERS_REDUCED_MOTION) {
  element.style.opacity = '1';
} else {
  animate(element, { opacity: [0, 1] }, { duration: 0.3 });
}
```

### Production Example

```javascript
// From hero_video.js
const PREFERS_REDUCED_MOTION = window.matchMedia(
  '(prefers-reduced-motion: reduce)'
).matches;

// Skip animation if reduced motion preferred
if (prefersReducedMotion) {
  window.scrollTo(0, 0);
  return;
}

// Otherwise use smooth scroll
window.lenis?.scrollTo(0, { duration: 1.2 });
```

## 5. ðŸŽ¯ FOCUS DETECTION

### Pattern

Distinguish keyboard focus from mouse focus for accessible styling.

### Body Class

| Class            | When Applied        | Purpose                      |
| ---------------- | ------------------- | ---------------------------- |
| `.using-keyboard`| User presses Tab    | Show focus outline           |
| (no class)       | User clicks/touches | Hide focus outline           |

### CSS Usage

```css
/* Show focus outline for keyboard users */
body.using-keyboard .input:focus {
  outline: 4px solid var(--_color-tokens---state--focused);
  outline-offset: 0;
}

/* Hide focus outline for mouse users */
body:not(.using-keyboard) .input:focus {
  outline: none;
}
```

### Implementation (input_focus_handler.js)

```javascript
let usingKeyboard = false;

// Detect keyboard navigation
document.addEventListener('keydown', (e) => {
  if (e.key === 'Tab') {
    usingKeyboard = true;
    document.body.classList.add('using-keyboard');
  }
});

// Detect mouse click
document.addEventListener('mousedown', () => {
  usingKeyboard = false;
  document.body.classList.remove('using-keyboard');
});

// Detect touch
document.addEventListener('touchstart', () => {
  usingKeyboard = false;
  document.body.classList.remove('using-keyboard');
}, { passive: true });
```

### Complete Focus States

```css
/* Keyboard focus - show outline */
body.using-keyboard .input:focus,
body.using-keyboard .w-input:focus {
  border-color: var(--_color-tokens---input-border--enabled);
  outline: 4px solid var(--_color-tokens---state--focused) !important;
  outline-offset: 0;
}

/* Mouse focus - hide outline */
body:not(.using-keyboard) .input:focus,
body:not(.using-keyboard) .w-input:focus {
  outline: none !important;
  outline-width: 0 !important;
}
```
