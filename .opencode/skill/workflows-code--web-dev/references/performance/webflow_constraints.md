# Webflow Performance Constraints

Platform-imposed limitations on performance optimization and available workarounds.

---

## 1. OVERVIEW

Webflow's platform imposes specific limitations on performance optimization. Understanding these constraints prevents wasted effort on impossible optimizations and helps focus on what IS controllable.

### Key Constraints

- **TypeKit/Adobe Fonts** - Synchronous loading, no async option
- **jQuery/webflow.js** - Auto-injected without defer
- **CSS Generation** - Single file, no critical CSS extraction
- **Custom Code** - Limited injection points with specific timing

---

## 2. TYPEKIT / ADOBE FONTS

### Limitation

- Loaded synchronously in `<head>`
- Webflow manages the embed code
- No option for async loading

### Impact

- Blocks HTML parsing for 500-2000ms
- Delays all subsequent resources

### Workaround

```html
<!-- Add preconnects in Project Settings > Custom Code > Head -->
<link rel="preconnect" href="https://use.typekit.net" crossorigin>
<link rel="preconnect" href="https://p.typekit.net" crossorigin>
```

### Potential Future Fix

- Audit if TypeKit fonts are actually used
- Consider self-hosting if Silka covers all needs

---

## 3. JQUERY / WEBFLOW.JS

### Limitation

- Auto-injected by Webflow platform
- Loads synchronously without defer
- Cannot modify loading behavior

### Impact

- Blocks page rendering
- Required for Webflow interactions

### Workaround

```html
<!-- Preconnect to jQuery CDN -->
<link rel="preconnect" href="https://d3e54v103j8qbb.cloudfront.net" crossorigin>
```

---

## 4. CSS GENERATION

### Limitation

- Single CSS file generated by Webflow
- Cannot split critical/non-critical
- Cannot extract above-fold CSS

### Impact

- Entire stylesheet blocks render
- No automatic critical CSS inlining

### Workaround

- Add custom critical CSS inline in head
- Async load custom below-fold CSS

---

## 5. CUSTOM CODE INJECTION POINTS

### Head Code (Project Settings)

- Runs after Webflow CSS link
- Before body content
- Good for: preconnects, preloads, meta tags

### Footer Code (Project Settings)

- Runs after all Webflow scripts
- Good for: analytics, custom scripts

### Page-Specific Code

- Available in Page Settings
- Overrides or supplements global code

---

## 6. SUMMARY TABLE

| Resource | Can Async? | Can Defer? | Workaround |
|----------|------------|------------|------------|
| TypeKit | No | No | Preconnect only |
| jQuery | No | No | Preconnect only |
| webflow.js | No | No | None |
| Main CSS | No | N/A | Custom critical CSS |
| Custom JS | Yes | Yes | Full control |
| Third-party | Yes | Yes | Full control |

---

## 7. WHAT IS CONTROLLABLE

Despite platform constraints, you CAN optimize:

- **Third-party scripts** (GTM, analytics) - Delay with requestIdleCallback
- **Custom JavaScript loading** - Full async/defer control
- **Custom CSS** (async pattern) - Preload with onload swap
- **Resource hints** (preconnect, preload) - Add via Project Settings
- **Image optimization** - WebP, lazy loading, srcset
- **Video loading strategy** - Poster images, lazy load, HLS
