10 // Extensibility

Extension Pattern Library

Determine the right customization path based on your architectural needs. Each pattern provides targeted capabilities while maintaining system integrity and readability.

When to use

You need to automate complex, multi-step workflows.

Best for integrating with external APIs, managing localized logic, or orchestrating multi-agent tasks.

Extension Skills

/create:skill

Skills are automatically detected by the routing module. They encapsulate business logic and prompt chains into a single, reusable command entity.

Terminal
$ /create:skill workflow-automation \
  --description "Orchestrates API deployments" \
  --type multi-step

When to use

You need to enforce specialized behavior and access scopes.

Ideal for defining specific operational boundaries, such as isolating a "Reviewer" or "Deployer" context.

Extension Roles

/create:role

Roles govern permissions and standard operating procedures. They allow the system to assume a persona with explicit constraints.

config/roles.yaml
role: security-auditor
permissions:
  - read: code
  - execute: static-analysis
constraints:
  - no-write-access

When to use

You need to standardize repetitive structural outputs.

Use this when generating boilerplates, documentation, or recurring architectural elements across the workspace.

System Templates

Template D

Update once, inherit everywhere. Templates define the structure and variables, allowing the platform to adapt seamlessly to your project's conventions.

!

Convention over configuration: Templates provide structure, but you own the content. Avoid hardcoding procedural logic directly into template files.
