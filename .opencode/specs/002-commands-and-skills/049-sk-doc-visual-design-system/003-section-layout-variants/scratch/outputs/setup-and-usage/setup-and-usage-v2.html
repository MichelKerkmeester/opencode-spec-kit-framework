Setup and Usage

Environment Configuration

Select your deployment target to generate the precise configuration schema and operational commands required for initialization.

Required Variables

PROVIDER_PROFILE Required
Defines the AI agent backend. Default to 'opencode' for local development.
LOCAL_STORAGE_PATH Required
Absolute path to the context repository. Must be writeable.
DEBUG_VERBOSITY Optional
Set to '3' for maximum trace output during local debugging.
opencode.json 
{
  "runtime": {
    "provider": "opencode",
    "storage": "./.gemini/context",
    "debug": 3
  },
  "validation": {
    "strict": false
  }
}
bash 
# 1. Initialize local workspace
opencode init --profile local

# 2. Verify connection
opencode verify --ping

Pipeline Variables

CI_TOKEN_AUTH Required
Secure injected secret for CI runner authentication.
NON_INTERACTIVE Required
Must be set to 'true' to prevent prompt blocking.
ARTIFACT_OUTPUT Optional
Directory to dump test execution summaries.
ci-config.json 
{
  "runtime": {
    "provider": "ci-runner",
    "headless": true
  },
  "validation": {
    "strict": true,
    "fail_fast": true
  }
}
yaml 
# Github Actions Example
steps:
  - name: Run Validations
    run: opencode check --ci
    env:
      AUTH_TOKEN: ${{ secrets.CI_TOKEN }}

Environment Variables

NODE_ENV Required
Must be strictly set to 'production'.
REMOTE_STATE_URL Required
Endpoint for distributed context synchronization.
prod-config.json 
{
  "runtime": {
    "provider": "production",
    "telemetry": false
  },
  "security": {
    "enforce_tls": true
  }
}
bash 
# 1. Pull remote state
opencode state pull

# 2. Start daemon
opencode daemon start --env prod
