Cookbook // Setup & Usage

Implementation & Diagnosis

A progressive guide to configuring and running the system. Each step is paired with an immediate diagnostic protocol to resolve common failures before moving forward.

Phase 01

Define Core Configuration

Establish provider endpoints, storage paths, and fallback behaviors in your local opencode.json.

opencode.json 
{
  "project": {
    "provider": "default-provider",
    "storage": "./data/context",
    "validation": true
  }
}

Diagnostic Protocol

SyntaxError: Unexpected token

Ensure trailing commas are removed. JSON strictly forbids them at the end of objects or arrays.

Storage path not writable

Run chmod 755 ./data/context to grant directory write permissions before proceeding.

Phase 02

Initialize Workspace

Run the initialization sequence to validate configuration schemas and scaffold required cache directories.

bash 
# 1. Verify environment schema against opencode.json
tool run setup:verify

# 2. Scaffold local cache and index architecture
tool run setup:init --force

Diagnostic Protocol

Command 'tool' not found

The CLI isn't in your PATH. Run npm link in the tool repository, or use the absolute binary path.

Schema Validation Failed

A required field from Phase 01 is missing or malformed. Double-check your opencode.json syntax.

Phase 03

Execute Core Workflow

With setup verified, run standard operations. Always generate a plan before applying changes to capture intent properly.

bash 
# 1. Generate execution plan context
tool run workflow:plan "update authentication middleware"

# 2. Apply changes and commit state ledger
tool run workflow:apply
tool run workflow:save

Diagnostic Protocol

Connection Timeout (504)

The provider API is unreachable. Switch to local fallback mode by appending --offline to your command.

Uncommitted Changes Block

The working directory must be clean before running apply. Commit or stash current git modifications.
