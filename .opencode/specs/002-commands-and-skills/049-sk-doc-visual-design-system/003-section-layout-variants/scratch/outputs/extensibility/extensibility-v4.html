10 // Extensibility

Adapt without disruption.

Convention over configuration. The extension model is designed to let you build on top of structural templates without rewriting the foundational layers. You own the content; the platform adapts to your project.

01
Isolated capabilities. Custom behaviors are encapsulated in distinct roles or skills, never polluting global state.
02
Declarative routing. The system automatically detects and routes to new extensions without manual registry updates.
03
Inheritable defaults. System templates act as fallbacks. Update the core once, and all extensions inherit the baseline.

Extension Skills

Auto-Detected

Generate a new specialized skill bounded to your workspace. The routing module will automatically map the command namespace upon generation.

# Create a new skill structure /create:skill my-custom-skill # The system generates: # .opencode/skill/my-custom-skill/ # ├── SKILL.md # ├── scripts/ # └── references/

Extension Roles

Define specific behavioral rules and access constraints using /create:role. Roles orchestrate how skills are permitted to interact.

Extension Commands

Add new CLI hooks via layered configuration files. Maintain clear separation between interface definitions and execution logic.

System Templates

Inheritance Layer

When extending visual outputs or documentation formats, derive from the core templates. This ensures your custom artifacts maintain full compatibility with the design system tokens.

import { baseConfig } from '@templates/core'; export const customExtension = { ...baseConfig, name: 'Extended Variant', overrides: [ 'theme', 'layout' ] };
