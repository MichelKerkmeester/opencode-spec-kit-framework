Operations Overview

Governance & Execution Split

This operational model segregates strict policy constraints from procedural execution steps. Governance dictates the immutable rules of the system, while procedures provide actionable, step-by-step workflows.

Governance Rules

Strict Enforced

Gate 3 Validation

Filesystem modifications explicitly require an active, validated SpecKit folder. Unscoped changes are rejected at the pre-execution boundary.

Automated

State Recording

Completion of any procedural workflow triggers an automatic implementation summary write. Level configurations dictate strictness thresholds.

Standard

Immutable Artifacts

Once an operation achieves complete status, corresponding records transition to read-only. Further revisions mandate a new operational phase.

Compliance Note

Policies are enforced programmatically by the orchestrator. Bypassing requires explicit, elevated user override flags.

Standard Execution Path

1

Initialize Scope

Bind the impending operation to a specific context boundary.

Terminal 
# Set workflow context
opencode init --scope section --target operations-overview
2

Apply & Validate

Execute changes while restricted by governance policy constraints.

Terminal 
# Run checks and apply
opencode apply --strict-validation

✓ Gate 3 Validation passed
✓ Scope lock maintained
3

Commit to Ledger

Finalize the operational phase, write records, and freeze artifacts.

Terminal 
# Finalize process
opencode record --freeze

Artifacts locked. Operation lifecycle complete.
